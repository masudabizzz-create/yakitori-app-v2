## 1. ログイン画面

### セッション期限の延長
- Supabase ダッシュボード → Authentication → Configuration → Sessions
- `JWT expiry` を `604800`（7日間）に変更
- `Persistent sessions` が ON になっているか確認

## 3. 発注推定（`/order`）

### 発注スケジュールを上部に表示
- 週選択カードの下に「今週の発注スケジュール」カードを追加
- 該当週の全スケジュール（通常＋イレギュラー）を表示

```
📦 今週の発注スケジュール

締め切り 5/25（月）→ 納品 5/27（水）
締め切り 5/27（水）→ 納品 5/29（金）
```

- `order_schedules` と `order_schedule_irregulars` から該当週のデータを取得して表示
- 複数ある場合は全件縦に並べる
- スケジュールがない場合は非表示

## 4. 運用管理（`/admin/ops`）— 串マスタタブ

### カテゴリ別の入力項目ハンドリング

カテゴリを選択した瞬間に関係ない項目を非表示にする。

| カテゴリ | 閾値1・2 | 仕込み量1・2 | 備考 |
|---------|---------|------------|------|
| レギュラー | 非表示 | 非表示 | 計算単位・曜日別理想在庫を表示 |
| つくね | 非表示 | 非表示 | 計算単位・曜日別理想在庫を表示 |
| スペシャル | 表示 | 表示 | |
| その他仕込み | 表示 | 表示 | |
| 前日仕込み | 表示 | 表示 | 昆布締め設定を追加表示 |
| 副産物 | 非表示 | 非表示 | |

### 詳細設定の追加（アコーディオン形式）

基本設定の下に「▼ 詳細設定を開く」を追加。

```
基本設定（常に表示）
  串名・カテゴリ・計算単位・閾値・仕込み量

▼ 詳細設定を開く（タップで展開）
  発注単位ラベル
  1発注単位あたりg
  1本あたり重量g
  歩留まり率
  対象コース（all_courses / specific_courses）
  対象コース配列
```

## 5. 運用管理（`/admin/ops`）— コース設定タブ

### 日曜ブーストの削除
- 「日曜上乗せ計算（日曜ブースト）」の設定項目を削除
- `settings` テーブルの `sunday_boost_enabled` カラムへの参照も削除
- 関連する表示・保存処理を全て除去

> ⚠️ `useInventoryCalc.ts` の計算ロジック内の日曜ブースト処理は触らない。
> あくまで設定画面からの表示・操作を削除するだけ。
